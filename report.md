
# Отчет по исследованию алгоритмов сортировки строк

## Цель работы

Целью данной работы является исследование производительности алгоритмов сортировки строк и сравнение стандартных алгоритмов сортировки с алгоритмами, адаптированными для строковых данных.

В рамках исследования проводился анализ следующих характеристик:

- времени выполнения алгоритмов;
- количества посимвольных сравнений.

# Этап 1. Подготовка тестовых данных

Для генерации тестовых данных был реализован класс `StringGenerator`.

## Используемый алфавит

При генерации строк использовался алфавит мощностью 74 символа:

- латинские буквы A-Z;
- латинские буквы a-z;
- цифры 0-9;
- специальные символы `!@#%:;^&*()-.`

## Параметры генерации

- длина строк: от 10 до 200 символов;
- размер массивов: от 100 до 3000 элементов;
- шаг изменения размера: 100.

## Типы тестовых массивов

### Случайные массивы

Строки расположены в случайном порядке.

### Обратно отсортированные массивы

Строки отсортированы в обратном лексикографическом порядке.

### Почти отсортированные массивы

Массивы получены из полностью отсортированного массива путем случайной перестановки нескольких пар элементов.

# Анализ результатов

## QUICK SORT

QUICK SORT демонстрирует высокую скорость работы на случайных массивах, однако чувствителен к степени упорядоченности входных данных.

На обратно отсортированных массивах наблюдается ухудшение производительности, что связано с выбором опорного элемента.

Количество посимвольных сравнений возрастает быстрее при увеличении размера массива.

## MERGE SORT

MERGE SORT показывает более стабильное поведение на различных типах данных.

Алгоритм менее чувствителен к степени упорядоченности массива и демонстрирует предсказуемую асимптотическую сложность.

Количество посимвольных сравнений растет более равномерно.

# Выводы

В результате исследования были получены следующие выводы:

1. Производительность алгоритмов сортировки строк существенно зависит от структуры входных данных.
2. QUICK SORT обеспечивает высокую скорость на случайных данных, однако может деградировать на почти отсортированных и обратно отсортированных массивах.
3. MERGE SORT демонстрирует более стабильное поведение и предсказуемую производительность.
4. Подсчет посимвольных сравнений позволяет более точно оценивать эффективность алгоритмов сортировки строк.
